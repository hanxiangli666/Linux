# IPtables
# IPtables 防火墙

> In this section we will learn about IPtables — the classic Linux firewall tool. We will cover how to inspect, add, delete, and organize firewall rules to control network traffic flowing into and out of your system.
>
> 本节我们将学习 IPtables——Linux 经典防火墙工具。我们将介绍如何检查、添加、删除和组织防火墙规则，以控制进出系统的网络流量。

---

## What is IPtables?
## 什么是 IPtables？

**IPtables** is a user-space utility that configures the Linux kernel's **Netfilter** packet filtering framework. It allows you to define rules that decide what happens to network packets: allow them, drop them, reject them, or redirect them.

**IPtables** 是一个用户空间工具，用于配置 Linux 内核的 **Netfilter** 数据包过滤框架。它允许你定义决定网络数据包命运的规则：允许、丢弃、拒绝或重定向它们。

> **Modern Alternative 现代替代方案:** On newer Linux systems (RHEL 8+, Ubuntu 20.04+), **`nftables`** and its frontend **`firewalld`** are gradually replacing IPtables. However, IPtables remains widely used and understanding it is essential for working with legacy systems and Docker networking.
>
> **现代替代方案：** 在较新的 Linux 系统（RHEL 8+、Ubuntu 20.04+）上，**`nftables`** 及其前端 **`firewalld`** 正逐渐取代 IPtables。但 IPtables 仍被广泛使用，理解它对于处理旧系统和 Docker 网络至关重要。

## Tables and Chains Architecture
## 表和链架构

IPtables organizes rules into **tables**, and each table contains **chains**. A chain is an ordered list of rules — packets are checked against each rule in order until one matches.

IPtables 将规则组织到**表**中，每个表包含**链**。链是规则的有序列表——数据包按顺序与每条规则进行匹配，直到找到匹配项。

### Table 1: FILTER (Default)
### 表1：FILTER（默认）

The most commonly used table — used for basic packet filtering.
最常用的表——用于基本数据包过滤。

| Chain 链 | Traffic Direction 流量方向 |
| --- | --- |
| **INPUT** | Packets destined for the local system 发往本地系统的数据包 |
| **FORWARD** | Packets routed through the system (not destined locally) 通过系统路由的数据包（不发往本地） |
| **OUTPUT** | Packets generated by the local system 由本地系统生成的数据包 |

### Table 2: NAT
### 表2：NAT

Used for Network Address Translation — modifying source/destination addresses.
用于网络地址转换——修改源/目标地址。

| Chain 链 | Purpose 用途 |
| --- | --- |
| **PREROUTING** | Alter packets immediately upon receipt 数据包到达时立即修改 |
| **OUTPUT** | Alter locally generated packets 修改本地生成的数据包 |
| **POSTROUTING** | Alter packets just before they leave 数据包离开前修改 |

### Table 3: MANGLE
### 表3：MANGLE

For specialized packet alteration (QoS, marking packets). Less commonly used.
用于专门的数据包修改（QoS、标记数据包）。不太常用。

## Viewing Rules
## 查看规则

```bash
# List all rules in the FILTER table
# 列出 FILTER 表中的所有规则
bob@devapp01:~$ sudo iptables -L

Chain INPUT (policy ACCEPT)
target     prot opt source               destination

Chain FORWARD (policy ACCEPT)
target     prot opt source               destination

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination
```

By default, all chains have a policy of **ACCEPT** (allow everything). Rules are added on top of this default.

默认情况下，所有链的策略为 **ACCEPT**（允许所有内容）。规则叠加在此默认值之上。

```bash
# List rules with line numbers (needed for deletion)
# 列出带行号的规则（删除时需要）
sudo iptables -L --line-numbers

# List rules with verbose output (shows packet/byte counters)
# 以详细输出列出规则（显示数据包/字节计数器）
sudo iptables -L -v

# List rules in numeric format (no DNS lookups, faster)
# 以数字格式列出规则（无 DNS 查找，更快）
sudo iptables -L -n

# Specify a table to list (e.g., NAT table)
# 指定要列出的表（例如 NAT 表）
sudo iptables -t nat -L
```

## Common IPtables Options
## 常用 IPtables 选项

| Option 选项 | Full Name 全名 | Description 描述 |
| --- | --- | --- |
| `-A` | `--append` | Append rule to end of chain 将规则追加到链末尾 |
| `-I` | `--insert` | Insert rule at position (default: top) 在指定位置插入规则（默认：顶部） |
| `-D` | `--delete` | Delete a specific rule 删除特定规则 |
| `-F` | `--flush` | Flush (delete all) rules in chain 刷新（删除所有）链中的规则 |
| `-P` | `--policy` | Set default chain policy 设置链的默认策略 |
| `-L` | `--list` | List all rules 列出所有规则 |
| `-p` | `--protocol` | Match by protocol (tcp, udp, icmp) 按协议匹配 |
| `-s` | `--source` | Match by source IP/network 按源 IP/网络匹配 |
| `-d` | `--destination` | Match by destination IP/network 按目标 IP/网络匹配 |
| `-i` | `--in-interface` | Match by incoming network interface 按入站网络接口匹配 |
| `-o` | `--out-interface` | Match by outgoing network interface 按出站网络接口匹配 |
| `--dport` | `--destination-port` | Match by destination port 按目标端口匹配 |
| `--sport` | `--source-port` | Match by source port 按源端口匹配 |
| `-j` | `--jump` | Target action (ACCEPT, DROP, REJECT, LOG) 目标动作 |

## Allowing Specific Connections
## 允许特定连接

```bash
# Allow incoming SSH (port 22) from a specific IP
# 允许来自特定 IP 的入站 SSH（端口 22）
sudo iptables -A INPUT -p tcp -s 172.16.238.187 --dport 22 -j ACCEPT

# Allow incoming HTTP (port 80) from that same IP
# 允许来自同一 IP 的入站 HTTP（端口 80）
sudo iptables -A INPUT -p tcp -s 172.16.238.187 --dport 80 -j ACCEPT

# Allow incoming HTTPS (port 443) from anywhere
# 允许来自任何地方的入站 HTTPS（端口 443）
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Allow all traffic on loopback interface (critical — don't block localhost!)
# 允许环回接口上的所有流量（关键——不要阻止本地主机！）
sudo iptables -A INPUT -i lo -j ACCEPT

# Allow established/related connections (for stateful packet inspection)
# 允许已建立/相关连接（用于有状态数据包检查）
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
```

After adding these rules, verify:
添加这些规则后，验证：

```bash
bob@devapp01:~$ sudo iptables -L
Chain INPUT (policy ACCEPT)
target     prot opt source               destination
ACCEPT     tcp  --  caleston-lp10  anywhere    tcp dpt:ssh
ACCEPT     tcp  --  caleston-lp10  anywhere    tcp dpt:http
ACCEPT     tcp  --  anywhere       anywhere    tcp dpt:https
ACCEPT     all  --  anywhere       anywhere

Chain FORWARD (policy ACCEPT)
target     prot opt source               destination

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination
```

## Blocking Traffic
## 阻止流量

```bash
# Drop ALL incoming connections not matched by earlier ACCEPT rules
# 丢弃所有未被之前 ACCEPT 规则匹配的入站连接
bob@devapp01:~$ sudo iptables -A INPUT -j DROP

# Block outgoing traffic to port 80 (HTTP)
# 阻止到端口 80（HTTP）的出站流量
bob@devapp01:~$ sudo iptables -A OUTPUT -p tcp --dport 80 -j DROP
```

> **Rule Order Matters! 规则顺序很重要！**
>
> IPtables processes rules **top-to-bottom** and stops at the first match. Always put specific ACCEPT rules **before** broad DROP rules. The `-A` (append) option adds rules to the **bottom** of the chain. Use `-I` (insert) to add rules at the **top** or a specific position.
>
> IPtables 从**上到下**处理规则，并在第一个匹配处停止。始终将特定的 ACCEPT 规则放在宽泛的 DROP 规则**之前**。`-A`（追加）选项将规则添加到链的**底部**。使用 `-I`（插入）将规则添加到**顶部**或特定位置。

```bash
# Insert a rule at position 1 (the very top) of INPUT chain
# 在 INPUT 链的第 1 位置（最顶部）插入规则
sudo iptables -I INPUT 1 -p tcp --dport 22 -j ACCEPT
```

## DROP vs REJECT
## DROP 与 REJECT 的区别

Both prevent packets from passing through, but behave differently:

两者都阻止数据包通过，但行为不同：

| Action 动作 | Behavior 行为 | Use Case 使用场景 |
| --- | --- | --- |
| **DROP** | Silently discards the packet. No response sent. 静默丢弃数据包，不发送响应 | External-facing rules — don't reveal firewall presence 面向外部的规则——不暴露防火墙的存在 |
| **REJECT** | Discards and sends an error (TCP RST or ICMP unreachable) back to sender 丢弃并向发送方发送错误响应 | Internal/trusted networks — give meaningful feedback 内部/受信任网络——提供有意义的反馈 |

```bash
# Using REJECT (sends ICMP error back)
# 使用 REJECT（发回 ICMP 错误）
sudo iptables -A INPUT -p tcp --dport 23 -j REJECT --reject-with icmp-port-unreachable
```

## Allowing Outbound Traffic
## 允许出站流量

```bash
# Allow HTTPS outbound to google.com (using -I to insert at top)
# 允许到 google.com 的 HTTPS 出站（使用 -I 插入到顶部）
bob@devapp01:~$ sudo iptables -I OUTPUT -p tcp -d google.com --dport 443 -j ACCEPT

```

## Deleting Rules
## 删除规则

### By Line Number (Recommended)
### 按行号删除（推荐）

```bash
# First, find the line number
# 首先，找到行号
bob@devapp01:~$ sudo iptables -L --line-numbers
Chain INPUT (policy ACCEPT)
num  target  prot opt source           destination
1    ACCEPT  tcp  --  caleston-lp10    anywhere    tcp dpt:ssh
2    ACCEPT  tcp  --  caleston-lp10    anywhere    tcp dpt:http
3    DROP    all  --  anywhere         anywhere

# Delete rule number 3 from INPUT chain
# 从 INPUT 链删除第 3 条规则
sudo iptables -D INPUT 3
```

### By Rule Specification
### 按规则规格删除

```bash
# Delete by specifying the exact rule (use -D instead of -A)
# 通过指定精确规则删除（使用 -D 代替 -A）
sudo iptables -D INPUT -p tcp -s 172.16.238.187 --dport 22 -j ACCEPT
```

### Flush All Rules
### 清空所有规则

```bash
# Flush all rules (careful — this removes ALL firewall protection!)
# 清空所有规则（小心——这会移除所有防火墙保护！）
sudo iptables -F
sudo iptables -F INPUT     # Only flush a specific chain 只清空特定链
```

## Advanced Matching
## 高级匹配

### Match Multiple Ports with `multiport`
### 使用 `multiport` 匹配多个端口

```bash
# Allow incoming traffic on ports 22, 80, and 443
# 允许端口 22、80 和 443 上的入站流量
sudo iptables -A INPUT -p tcp -m multiport --dports 22,80,443 -j ACCEPT
sudo iptables -A OUTPUT -p tcp -m multiport --sports 22,80,443 -j ACCEPT
```

### Block Ping Requests (ICMP)
### 阻止 Ping 请求（ICMP）

```bash
# Block all incoming ICMP (ping) on interface eth0
# 阻止 eth0 接口上的所有入站 ICMP（ping）
sudo iptables -A INPUT -p icmp -i eth0 -j DROP

# Block only ICMP echo requests (ping) — more targeted
# 只阻止 ICMP 回显请求（ping）——更有针对性
sudo iptables -A INPUT -p icmp --icmp-type echo-request -j DROP
```

### Block by MAC Address
### 按 MAC 地址阻止

```bash
# Block a specific MAC address
# 阻止特定 MAC 地址
sudo iptables -A INPUT -m mac --mac-source 0e:Ds:8n:mq:00:de -j DROP
```

### Match by Connection State
### 按连接状态匹配

```bash
# Only allow responses to connections we initiated (stateful firewall)
# 只允许我们发起的连接的响应（有状态防火墙）
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
sudo iptables -A INPUT -m state --state NEW,INVALID -j DROP
```

## Making Rules Persistent
## 使规则持久化

By default, IPtables rules are **lost on reboot**. To persist them:

默认情况下，IPtables 规则在**重启后丢失**。要持久化它们：

```bash
# Ubuntu/Debian — save rules
sudo apt install iptables-persistent
sudo netfilter-persistent save

# RHEL/CentOS
sudo service iptables save
# Rules saved to /etc/sysconfig/iptables

# Manual save/restore
sudo iptables-save > /etc/iptables/rules.v4
sudo iptables-restore < /etc/iptables/rules.v4
```

## Practical Example: Basic Server Hardening
## 实践示例：基本服务器加固

A typical secure setup for a web server:

Web 服务器的典型安全设置：

```bash
# 1. Allow loopback interface 允许环回接口
sudo iptables -A INPUT -i lo -j ACCEPT

# 2. Allow established/related connections 允许已建立/相关连接
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 3. Allow SSH from management network only 只允许来自管理网络的 SSH
sudo iptables -A INPUT -p tcp -s 10.0.1.0/24 --dport 22 -j ACCEPT

# 4. Allow HTTP and HTTPS from anywhere 允许来自任何地方的 HTTP 和 HTTPS
sudo iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

# 5. Drop everything else 丢弃其他所有内容
sudo iptables -A INPUT -j DROP

# 6. Save the rules 保存规则
sudo netfilter-persistent save
```
