### Exercise 1 — Check the Exact Kernel Version
### 练习 1 — 检查精确的内核版本

```bash
$ uname -r
4.15.0-88-generic
```

**Explanation / 说明**: `uname -r` prints the **kernel release version** — the most commonly used option when you need the kernel version string.

`uname -r` 打印**内核发行版本**——这是需要内核版本字符串时最常用的选项。

**Other useful `uname` options / 其他有用的 `uname` 选项:**
```bash
$ uname        # Just "Linux"
$ uname -r     # Kernel version: 4.15.0-88-generic
$ uname -n     # Hostname: ubuntu
$ uname -m     # Machine hardware: x86_64
$ uname -o     # Operating system: GNU/Linux
$ uname -a     # All of the above combined / 以上所有信息的组合
```

### Exercise 2 — Identify the Kernel Version Number
### 练习 2 — 识别内核版本号

**Question / 问题**: In the version string `4.15.0-88-generic`, what is the **kernel version**?

**Answer / 答案**: **`4`** — the first digit before the first dot.

```
4  .  15  .  0   -  88  -  generic
│     │      │      │       └── Distro flavor / 发行版变体
│     │      │      └── ABI version / ABI 版本
│     │      └── Patch level / 补丁级别
│     └── Major version (significant new features) / 主版本（重要新功能）
└── Kernel version / 内核版本号
```

> **Historical note / 历史背景**: In older Linux kernel numbering (before 2.6), even major numbers meant "stable release" and odd numbers meant "development/testing". This convention was dropped after the 2.6 kernel. Modern kernels simply increment the version number linearly.
>
> 在旧版 Linux 内核编号中（2.6 之前），偶数主版本号表示"稳定版"，奇数表示"开发/测试版"。这一惯例在 2.6 内核之后被废弃。现代内核只是线性递增版本号。

### Exercise 3 — Identify the Major Version Number
### 练习 3 — 识别主版本号

**Question / 问题**: In the version string `4.15.0-88-generic`, what is the **major version number**?

**Answer / 答案**: **`15`** — the second number, after the first dot.

```bash
# You can extract just the major version with awk / 可以用 awk 提取主版本号
$ uname -r | cut -d. -f2
15

# Or with awk / 或用 awk
$ uname -r | awk -F. '{print $2}'
15
```

### Exercise 4 — Print Kernel Messages
### 练习 4 — 打印内核消息

**Question / 问题**: Which command prints messages generated by the kernel?

**Answer / 答案**: `dmesg`

```bash
$ dmesg

# More useful variants / 更有用的变体
$ dmesg -T              # with human-readable timestamps / 带易读时间戳
$ dmesg | grep -i usb   # filter USB messages / 过滤 USB 消息
$ dmesg | grep -i error # filter error messages / 过滤错误消息
$ dmesg | tail -20      # last 20 messages / 最后 20 条消息
$ dmesg -w              # follow new messages in real time / 实时跟踪新消息
```

> **Why `dmesg` matters / 为什么 `dmesg` 很重要**: This is often the **first place to look** when a hardware device isn't working. Messages include:
> - Hardware initialization during boot / 启动时的硬件初始化
> - Driver loading success/failure / 驱动程序加载成功/失败
> - Disk I/O errors / 磁盘 I/O 错误
> - USB device plug/unplug events / USB 设备插拔事件
> - Memory errors / 内存错误

### Exercise 5 — Count Block Devices of Type Disk
### 练习 5 — 统计磁盘类型的块设备数量

**Question / 问题**: How do you list and count all block devices of type `disk`?

```bash
$ lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0    50G  0 disk
├─sda1   8:1    0    49G  0 part /
└─sda5   8:5    0   975M  0 part [SWAP]
sdb      8:16   1    30G  0 disk
```

**Count only disk-type devices / 只统计磁盘类型的设备:**
```bash
$ lsblk | grep disk
sda    8:0    0   50G  0 disk
sdb    8:16   1   30G  0 disk

# Count them / 统计数量
$ lsblk | grep -c disk
2
```

**Understanding device types / 理解设备类型:**
- `disk` — a physical or virtual disk / 物理或虚拟磁盘
- `part` — a partition on a disk / 磁盘上的分区
- `rom` — a CD/DVD-ROM / 光驱
- `loop` — a loop device (used for mounting .iso files, snap packages, etc.) / 回环设备（用于挂载 .iso 文件、snap 包等）
- `lvm` — a Logical Volume Manager volume / 逻辑卷管理器卷

### Exercise 6 — Check Total Physical CPU Cores
### 练习 6 — 检查总物理 CPU 核心数

```bash
$ lscpu
Architecture:        x86_64
CPU(s):              4
Thread(s) per core:  2
Core(s) per socket:  2
Socket(s):           1
```

**Calculation / 计算:**
```
Total Physical Cores = Core(s) per socket × Socket(s)
总物理核心数 = 每插槽核心数 × 插槽数
             = 2 × 1 = 2 physical cores

Total Logical CPUs = CPU(s) = 4
(2 cores × 2 threads per core via Hyper-Threading)
```

**Quick calculation with `lscpu` / 用 `lscpu` 快速计算:**
```bash
# Extract and compute / 提取并计算
$ lscpu | grep -E "^Core|^Socket"
Core(s) per socket:  2
Socket(s):           1

# One-liner to calculate total physical cores / 计算总物理核心数的单行命令
$ echo $(($(lscpu | awk '/Core\(s\) per socket/{print $NF}') * $(lscpu | awk '/Socket\(s\)/{print $NF}')))
2
```

### Exercise 7 — Check Total Online Memory
### 练习 7 — 检查总在线内存

```bash
$ lsmem
RANGE                                  SIZE  STATE REMOVABLE BLOCK
0x0000000000000000-0x000000007fffffff    2G online       yes   0-15
0x0000000100000000-0x000000047fffffff   14G online       yes  32-143

Memory block size:       128M
Total online memory:      16G
Total offline memory:      0B
```

**Answer / 答案**: Look for the **`Total online memory`** line — in this example, `16G`.

查找 **`Total online memory`** 行——在本例中为 `16G`。

**Alternative commands / 替代命令:**
```bash
# Summary only / 只显示摘要
$ lsmem --summary

# Using free / 使用 free
$ free -h
              total
Mem:           16Gi

# Using /proc/meminfo / 使用 /proc/meminfo
$ grep MemTotal /proc/meminfo
MemTotal:       16384000 kB
```

### Bonus — Kernel Module Commands
### 附加题 — 内核模块命令

```bash
# List all currently loaded modules / 列出所有当前加载的模块
$ lsmod

# Get information about a specific module / 获取特定模块的信息
$ modinfo bluetooth
$ modinfo ext4

# Load a module / 加载模块
$ sudo modprobe bluetooth

# Remove a module / 移除模块
$ sudo modprobe -r bluetooth

# Load module at boot (make it permanent) / 在启动时加载模块（使其永久生效）
$ echo "bluetooth" | sudo tee -a /etc/modules

# Check if a module is loaded / 检查模块是否已加载
$ lsmod | grep bluetooth
```

**Understanding `lsmod` output / 理解 `lsmod` 输出:**
```bash
$ lsmod
Module                  Size  Used by
bluetooth             614400  5 bnep,btrtl,btbcm,btintel,btusb
rfkill                 28672  5 bluetooth,cfg80211
crc16                  16384  1 bluetooth
```
- `Module` — module name / 模块名
- `Size` — memory used by the module in bytes / 模块占用的内存（字节）
- `Used by` — other modules that depend on this one / 依赖此模块的其他模块
